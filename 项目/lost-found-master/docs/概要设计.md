社区失物招领APP分为6个模块：用户管理、社区管理、发布管理、认领管理、评论留言、消息推送
模块内部构成
2.1 用户管理模块
1. 用户注册登录
（1）用户注册
（业务逻辑：新用户创建账户。）
填写注册表单信息：
{
用户ID：String（系统自动生成），
用户名：String，
密码：String（加密存储），
手机号：String（可选），
头像：File（可选，默认头像），
注册时间：DateTime（系统自动获取）
}
2. 用户信息管理
（2）用户信息管理
（1）查看个人信息 通过当前用户ID查询用户表，显示用户名、手机号、头像、注册时间等基本信息。
（2）编辑个人信息 填写修改表单信息：
{
用户名：String，
手机号：String，
头像：File
}
点击保存后，通过用户ID更新用户表信息，显示修改结果。
2.2 社区管理模块
 
1. 社区创建
（1）创建社区 （业务逻辑：管理员创建新社区。） 
填写社区信息表单：
{
社区ID：String（系统自动生成），
社区名称：String，
社区类型：Enum（校园/小区/公司/车站/机场），
地址：String，
创建者ID：String（当前用户ID），
创建时间：DateTime（系统自动获取）
}
2. 社区加入
（1）搜索社区 根据社区名称、类型、地址等条件搜索可加入的社区列表。
（2）申请加入 选择目标社区，提交加入申请：
{
用户ID：String（当前用户），
社区ID：String（目标社区），
加入时间：DateTime（系统自动获取）
}
后直接加入社区。
3. 社区更新
（1）社区信息编辑 社区创建者权限：修改社区名称、类型、地址等基本信息。
2.3 发布管理模块
 
1. 社区选择
 
（1）社区列表显示 显示用户已加入的所有社区列表，包含社区名称、类型、成员数等信息。
（2）切换当前社区 选择特定社区作为当前所在社区版块，后续发布和搜索操作在该社区内进行。
2. 信息搜索
 
（1）关键词搜索 （业务逻辑：在当前社区内搜索相关失物信息。）
输入搜索关键词，在物品名称、物品描述中进行模糊匹配，返回相关求助信息列表。
（2）条件筛选 根据信息类型（寻物/寻主）、发布时间、状态标记等条件筛选搜索结果。
3. 求助发布
 
（1）发布前搜索提醒 （业务逻辑：避免重复发布，提高匹配效率。）
用户输入物品名称后，自动搜索相关信息并提醒是否已有相似物品信息。
（2）信息录入 填写求助信息表单：
{
求助ID：String（系统自动生成），
物品名称：String，
物品描述：String，
物品图片：File[]（可选，支持多张图片），
信息类型：Enum（寻物/寻主），
丢失时间：DateTime，
丢失地点：String，
联系方式：String（可选），
暂存地点：String（寻主时必填，寻物时不填），
发布者ID：String（当前用户），
社区ID：String（当前选择社区），
状态标记：Enum（寻找中/已找到，默认寻找中）
}
（3）信息发布 验证必填项，上传图片，发布成功后显示在求助列表中。
4. 求助更新
（1）信息修改 发布者可以修改求助信息，更新物品名称、描述、图片、联系方式等。
（2）状态更新 发布者可以将状态从"寻找中"更新为"已找到"。
2.4 认领管理模块	
 
1. 认领登记
（1）认领申请 （业务逻辑：用户线下领走失物后，需在线上同步状态。）
填写认领信息：
{
认领ID：String（系统自动生成），
认领者ID：String（当前用户），
求助ID：String（目标求助ID），
认领时间：DateTime（系统自动获取），
认领者手机号：String
}
（2）认领处理 更新求助状态为"已找到"。
2.5 评论留言模块
 
1. 评论功能
（1）发表评论 （业务逻辑：用户对求助信息进行评论交流。）
在物品详情页填写评论内容：
{
评论ID：String（系统自动生成），
评论内容：String，
评论图片：File[]（可选），
评论者ID：String（当前用户），
求助ID：String（目标求助信息），
评论时间：DateTime（系统自动获取）
}
（2）评论展示 按时间顺序显示评论列表，包含评论者头像、用户名、评论内容、评论时间。
2. 私信功能
（1）发送私信 （业务逻辑：用户间私下交流。）
填写私信内容：
{
私信内容：String，
私信图片：File[]（可选），
发送者ID：String（当前用户），
接收者ID：String（目标用户），
发送时间：DateTime（系统自动获取）
}
（2）私信详情页 显示与特定用户的私信对话历史，支持图片显示。
2.6 消息推送模块
 
1. 消息通知
（1）评论通知 当有人在用户的求助详情页下方发评论时，在通知列表向用户通知，用户点击后可跳转到评论处。
（2）私信通知 当有人向用户发送私信时，在通知列表向用户通知，用户点击后可跳转到与那人的私信页面。
2. 消息管理
（1）消息列表 （业务逻辑：集中管理所有消息通知。）
按消息类型分类显示：
{
消息ID：String，
消息类型：Enum（评论/私信），
接收者ID：String（当前用户），
消息内容：String（含图片），
发送时间：DateTime，
阅读状态：Boolean（未读/已读）
}
（2）消息分类管理 支持按消息类型筛选，显示未读消息数量，标记已读/未读状态。
